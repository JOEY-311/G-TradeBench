# 跨境食品合规能力技能

本技能不绑定具体任务名称，而是提供一组**可自由组合的底层能力**。
面对任何新问题，先识别需要哪些能力，再按需加载对应的参考文档。

---

## 能力地图

| 能力 | 适用场景 | 参考文档 |
|------|---------|---------|
| **模型调用** | 批量调用 LLM、断点续传、LLM-as-Judge、清洗输出 | `capabilities/llm-call.md` |
| **合规判断** | 判断食品是否违规、提取审查结论、规则打分 | `capabilities/compliance-check.md` |
| **时序推理** | 判断法规草案/已生效状态、提取生效日期、F值计算 | `capabilities/temporal-reason.md` |
| **场景问答** | 回答进出口专业咨询、引用法规、D值评分 | `capabilities/scenario-qa.md` |
| **RAG 增强** | 将本地法规文档注入 prompt 提升准确率 | `capabilities/rag.md` |
| **CoT 增强** | 为 prompt 添加分步推理，提升推理质量 | `capabilities/cot.md` |

## 决策逻辑：拿到请求后怎么做

**第一步：识别目标**
用户最终想要的是什么？
- 生成一批模型输出 → 必然需要 **模型调用**
- 检验输出质量 → 规则打分见对应任务文档，LLM-judge 见 **模型调用**
- 改进输出质量 → 考虑 **RAG 增强** 或 **CoT 增强**

**第二步：识别任务性质**
输出的内容属于哪种判断类型？
- 判断食品能否通关、是否超标 → **合规判断**
- 判断法规处于什么状态、何时生效 → **时序推理**
- 回答进出口咨询、提出操作方案 → **场景问答**
- 以上组合，或全新的任务类型 → 按需组合多个能力

**第三步：加载对应参考文档**
只读需要的文档，不必全部加载。

**第四步：查阅项目上下文（按需）**
如果需要文件路径、已有模型记录、数据格式：
- `project/paths.md` — 所有目录和文件路径
- `project/models.md` — 已测试模型及得分
- `project/data-formats.md` — CSV/JSONL 字段说明

## 通用工作流程模板

无论什么任务，流程大致如下：

```
1. 明确输入数据在哪里，格式是什么
        ↓
2. 设计 prompt（参考对应能力文档）
        ↓
3. 决定是否加 RAG / CoT 增强
        ↓
4. 调用模型，保存输出（含断点续传）
        ↓
5. 评测输出质量（规则 or LLM-judge）
        ↓
6. 解读结果，决定是否调整策略
```

## 快速判断：RAG 和 CoT 什么时候用

| | 合规判断 | 时序推理 | 场景问答 |
|---|---|---|---|
| **RAG** | ✅ 显著提升（注入限量数据）| ⚠️ 慎用（中文文档干扰英文推理）| ✅ 小幅提升（D2 法规引用）|
| **CoT** | ⚠️ 需严格控制输出格式 | ✅ 提升 F1/F2 | ✅ 基本无负面影响 |
| **RAG+CoT** | ⚠️ 结论提取难度最高 | ✅ 与单 CoT 相近 | ✅ 与单 RAG/CoT 相近 |

详见各能力文档中的"注意事项"章节。
