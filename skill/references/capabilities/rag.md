# RAG 增强能力

将本地法规文档检索结果注入 prompt，为模型提供精确的外部知识支撑。

---

## 核心思路

模型的参数化知识对于具体限量数值可能不准确或过时。
RAG 的价值在于：在调用时动态注入**当前有效的法规原文片段**，
让模型基于真实文档作判断，而不是依赖记忆。

---

## 索引构建

### 支持的文件格式

| 格式 | 读取方式 |
|------|---------|
| PDF | `pdfplumber`（优先）→ `PyPDF2`（降级）|
| DOCX | `python-docx`（优先）→ 直接解 ZIP XML（降级）|
| TXT | 直接读取，多编码尝试 |

### 文本切块参数

```python
CHUNK_SIZE    = 600   # 每块字符数（中文约 200-300 字）
CHUNK_OVERLAP = 80    # 相邻块重叠（保证上下文连续性）
MAX_PER_FILE  = 120   # 每文件最多保留块数（防止超大文件撑爆索引）
```

### 国家标签分配

按文件夹路径自动判断：
```python
COUNTRY_MAP = {
    '中国': ['中国'],
    '日本': ['日本'],
    '韩国': ['韩国'],
    '美国': ['美国'],
    '德法': ['德国', '法国'],   # 德法目录同时服务两国
}
```

### 索引格式

```json
[
  {
    "country": "日本",
    "source": "日本\\法规\\厚生劳动省\\食品卫生法.pdf",
    "text": "...文本块内容..."
  }
]
```

保存为单个 JSON 文件（通常 5~20 MB），启动时一次性加载到内存。

## 检索逻辑

### 关键词评分检索（零依赖，无需向量库）

```python
def retrieve_local(country, keywords, index, top_n=4):
    # 第一步：按国家过滤
    country_chunks = [c for c in index if c['country'] == country]
    if not country_chunks:
        return ''   # 该国无数据，回退到硬编码知识

    # 第二步：按关键词命中数排序
    scored = sorted(
        [(c, sum(1 for kw in keywords if kw in c['text']))
         for c in country_chunks],
        key=lambda x: -x[1]
    )

    # 第三步：取命中至少 1 个关键词的 top-N 块
    top = [item[0] for item in scored[:top_n] if item[1] > 0]
    if not top:
        return ''   # 关键词全未命中，回退

    header = f'【{country}进口法规参考（来源：本地文件）】'
    body = '\n---\n'.join(
        f'[{c["source"].split(chr(92))[-1][:35]}]\n{c["text"]}'
        for c in top
    )
    return header + '\n' + body
```

### 关键词选择建议

| 任务 | 推荐关键词 |
|------|-----------|
| 合规审查 | `['铅', '镉', '农药', '限量', '污染物', '重金属', '最大残留量']` + 食品品类词 |
| 标准对齐-限量类 | `['限量', 'MRL', '最大残留量', '铅', '镉', '农药', '污染物']` |
| 标准对齐-添加剂 | `['食品添加剂', '准入', '正面清单', 'E-number', 'GRAS', '使用基准']` |
| 标准对齐-标签 | `['标签', '标识', '过敏原', '营养', '成分表']` |
| 标准对齐-程序 | `['注册', '进口', '申报', '检验检疫', '口岸']` |

## 注入到 Prompt

将检索结果插入在食品档案或问题之前：

```
你现在是一位资深的跨国食品合规与通关专家。
{rag_block}          ← 检索到的法规文本块（可能为空）
【拟进口食品档案】：
{profile}
```

`rag_block` 不为空时格式：
```
【日本进口法规参考（来源：本地文件）】
[食品卫生法.pdf]
...法规原文片段...
---
[農薬の成分.pdf]
...另一个片段...
```

## 兜底机制

当以下情况发生时，回退到硬编码的简要知识：
- 索引文件不存在
- 该国在索引中没有数据
- 关键词与任何块都不匹配（全部命中数为 0）

硬编码知识可以作为最基础的背景，而不是正式的法规引用：
```python
FALLBACK = {
    '日本': '铅≤0.1 mg/kg；镉≤0.05 mg/kg；农药肯定列表制度，未列明≤0.01 mg/kg',
    '中国': 'GB 2762-2022：铅(畜禽肉)≤0.2；镉(畜肉)≤0.1；GB 2763-2021：农药MRL',
    # ...
}
```

## 注意事项

- **语言匹配很重要**：中文法规文档对中文任务有效，但对英文时序推理任务可能产生干扰。
  实测表明，将中文法规 RAG 注入到英文时序推理任务中，F2 从 95% 跌至 75%。
- **块数控制**：top_n 建议设为 3~4，过多会稀释 prompt 中的有效信息。
- **去重**：索引构建时用 MD5 对每块文本去重，避免重复文件（如 _origin.pdf）被计入两次。
- **大文件**：单文件超过 MAX_PER_FILE 块时，均匀采样（头 60% + 尾 40%），
  保留核心条款区域的同时不丢弃末尾附件表格。
